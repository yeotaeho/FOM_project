# 폼미쳤다 (FOM) - 프로젝트 컨텍스트

## 1. 프로젝트 개요

### 1.1 기본 정보
- **프로젝트명:** 폼미쳤다 (FOM)
  - "너 오늘 춤 폼 미쳤다" 할 때 그 폼! 진단받고 폼 확인하는 곳
- **팀명:** Hi-Six
  - "인사는 가볍게, 퀄리티는 높게(High)! 완벽한 호흡으로 뭉친 6명의 인재들, Hi-Six"
- **개발 기간:** 1주일 MVP
- **타겟 사용자:** 10대 청소년

### 1.2 한 줄 소개
> AI 비전과 LLM 기술을 활용해 10대 청소년의 스트릿 댄스 동작을 분석·평가하고, 맞춤형 피드백과 진로 가이드를 제공하는 앱 기반 MVP 플랫폼입니다.

**캐치프레이즈:**
> **"방구석이 나만의 무대로, AI가 찾아주는 나의 댄스 DNA"**

## 2. 프로젝트 목적

### 2.1 핵심 목표
1. **정밀한 동작 분석:** AI 비전 기술을 활용해 10대 청소년의 스트릿 댄스 동작을 전문가 영상과 정밀하게 비교·평가
2. **맞춤형 피드백:** 분석 데이터를 기반으로 맞춤형 교정 피드백 제공
3. **진로 지도:** LLM 중심의 실질적인 진로 지도 가이드 제공
4. **신속한 MVP:** 1주일 내 웹 MVP 플랫폼 구축

### 2.2 기대 효과

#### 사용자 관점
- **교육 격차 해소:** 비싼 학원에 가지 않아도 어디서나 전문가급 피드백 수강 가능
- **심리적 장벽 완화:** 비대면 AI 코칭과 10대 맞춤형 피드백으로 부담 없이 자신감 형성
- **구체적 진로 설계:** 내 춤 실력을 데이터로 확인하고 막연한 꿈을 실질적 진로로 연결

#### 기술적 효과
- **무용 데이터 정량화:** 주관적인 감의 영역을 AI 비전 기반의 객관적 지표로 전환
- **스트릿 댄스 특화 분석:** 고유의 동적 리듬과 고난도 테크닉을 정밀 분석하는 알고리즘 정립
- **하이브리드 AI MVP 검증:** Vision(분석)과 LLM(가이드)을 결합한 웹 서비스를 1주일 내 신속 구현

#### 확장 가능성
- **장르 및 도메인 확장:** K-POP 댄스, 발레, 홈트레이닝 등 신체 움직임 분석 산업 전반으로 확대
- **B2B 비즈니스 연계:** 기획사 비대면 오디션 시스템 및 오프라인 학원 관리 솔루션으로 납품
- **글로벌 플랫폼화:** 유저 데이터를 축적해 글로벌 인재 발굴(스카우팅) 및 댄서 커뮤니티로 성장

## 3. 문제 정의

### 3.1 문제 상황
- **전문적 피드백 부재:** 10대 청소년들이 혼자 춤을 연습할 때, 전문가의 안무와 자신의 동작을 정밀하게 비교·교정하기 어려움
- **스트릿 댄스 특화 지표 부족:** 정형화되지 않은 스트릿 댄스 고유의 동적 리듬과 테크닉을 객관적으로 평가할 기준이 없음
- **10대 맞춤형 진로 컨설팅 결여:** 춤에 소질이 있어도 이를 실제 미래 진로와 구체적으로 연결해 줄 실질적인 가이드가 부족함

### 3.2 기존 해결 방식의 한계
- **PT/코치:** 비용이 높고 시간/장소 제약이 큼
- **유튜브 영상:** 일방향 학습으로 개인화된 피드백 불가능
- **기존 댄스 앱:** 단순 동작 카운팅 수준으로 정밀한 자세 분석 부재

### 3.3 AI가 해결해야 할 핵심 문제
1. **동작 유사도 정량화:** 전문가와 사용자의 동작을 객관적인 수치로 비교
2. **개인별 체형 차이 반영:** 키, 팔다리 길이 등 체형 차이를 정규화하여 공정한 평가
3. **스트릿 댄스 특화 평가:** ROM(관절 가동 범위), Power(순간 가속도), Isolation(부위별 독립성), Rhythm(박자 정확도), Creativity(독창성), Accuracy(전문가 대비 정확도) 등 다차원 분석
4. **실시간 피드백 생성:** 분석 결과를 기반으로 LLM이 10대 눈높이에 맞는 구체적이고 동기부여적인 피드백 제공

## 4. 핵심 기능

### 4.1 사용자 여정 (User Journey)
```
1. 영상 업로드 (MP4/MOV 등)
   ↓
2. AI 분석 실행 (서버에서 자동 처리)
   - MediaPipe로 3D 랜드마크 추출
   - 데이터 스무딩 및 정규화
   - 6개 차원 채점
   ↓
3. 결과 확인
   - 종합 점수 및 등급
   - 6개 항목별 세부 점수
   - 시각화 (관절 움직임 오버레이)
   ↓
4. LLM 피드백 조회
   - 10대 맞춤형 언어로 작성된 개선 방안
   - 진로 가이드 (댄서/안무가/강사 등)
```

### 4.2 기능 상세

#### Phase 1: 데이터 추출 (현재 구현 완료)
- **영상 업로드:** FastAPI 엔드포인트 `/video/extract`
- **포즈 추정:** MediaPipe Pose로 33개 랜드마크 추출
- **데이터 정제:** 보간(interpolation) + 이동평균 필터링
- **정규화:** Mid-Hip 기준 이동 + Torso Length 스케일링
- **표준 JSON 출력:** fps, total_frames, 프레임별 landmarks + normalized_landmarks

#### Phase 2: 채점 모듈 (개발 예정)
- **ROM (Range of Motion):** 관절별 최대/최소 각도 범위
- **Power:** 순간 가속도 및 에너지 방출
- **Isolation:** 특정 신체 부위만 움직이는 독립성
- **Rhythm:** 음악 BPM 대비 동작 타이밍 정확도
- **Creativity:** 전문가 영상과의 차별화된 독창적 요소
- **Accuracy:** 전문가 영상과의 정규화된 좌표 유사도

#### Phase 3: LLM 피드백 (개발 예정)
- 6개 채점 결과를 프롬프트로 전달
- 10대 친화적 톤앤매너로 개선 방안 제시
- 강점 항목 기반 진로 추천 (예: Rhythm 높음 → 프리스타일 특화)

#### Phase 4: 시각화 (개발 예정)
- 프레임별 랜드마크 오버레이
- 전문가 vs 사용자 스켈레톤 비교
- 점수 레이더 차트

- Tailwind CSS

## 6. 아키텍처 원칙

### 6.1 Hub-Spoke 패턴
```
domain1/
├── hub/              ← 공통 코어 (비디오 추출, 채점 엔진)
│   ├── services/     ← 비즈니스 로직
│   ├── repositories/ ← 데이터 접근
│   └── mcp/          ← Model Context Protocol
├── spokes/           ← 확장 기능
│   ├── agents/       ← LLM 에이전트
│   └── infra/        ← 외부 연동
└── models/           ← 데이터 스키마
```

### 6.2 설계 철학
1. **One-Time Processing:** 영상은 1회만 처리하고 표준 JSON을 6개 채점 함수가 재사용
2. **Parallel Development:** 각 채점 함수는 독립적으로 개발 가능
3. **Scalability:** 향후 장르 확장 시 새 domain 추가 (domain2: K-POP, domain3: Ballet 등)

## 8. 비기능 요구사항

### 8.1 성능
- 영상 처리 시간: 1분 영상 기준 30초 이내 (목표)
- 동시 처리: 초기 MVP는 순차 처리, 추후 비동기 큐(Celery) 도입

### 8.2 보안
- 업로드 파일 크기 제한: 500MB
- 허용 확장자: mp4, mov, avi, mkv, webm
- 임시 파일 자동 삭제

### 8.3 확장성
- 채점 함수 추가 시 기존 추출 모듈 수정 불필요
- 새 장르 추가 시 domain2, domain3 등 독립적으로 구성
